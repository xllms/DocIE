Introduction

Overview: Universal Document-level Information Extraction Task.

We propose the Universal Document-level Information Extraction challenge on the this platform. DocIE aims to detect entities, the mention of the entities and relations from document-level corpus at the same time. Therefore, the DUIE is defined as: Given a paragraph consisting of a series of sentences, the goal is to extract structured information comprising four elements: 1) the entities type, 2) the relation triples.

The challenge contains three subtasks as following:

Challenge Task Definition and Metrtics

Task 1: Named Entity Recognition.

Task-1: Named Entity Recognition (NER) aims to identify named entities in the given paragraph and classify them into corresponding categories. Participants are required to develop models that can accurately extract the named entities and its type. A named entity is considered correctly extracted if it matches the gold standard entity and its corresponding type. The evaluation metric for this sub-task is the F1 score. Unlike traditional sentence-level NER tasks, participants are also required to detemine the mention span for each entity within the paragraph.

Task 2: Relation Extraction.

Task-2: Relation Extraction (RE) aims to identify the relation between two entities in the given paragraph. Participants are required to develop models that can accurately identify the relation between two extracted entities. A relation is considered correctly identified if it matches the gold standard relation. The evaluation metric for this sub-task is the F1 score. Unlike traditional sentence-level RE task, the entities in a relation triple may across multiple sentences, and the relations between entities may also be inferred from the context include multiple sentences.

Evaluation

The F1 score is computed using precision (P) and recall (R), which are calculated as follows:

P = TP / ( TP + FP )

R = TP / ( TP + FN )

F1 = 2 * P * R / ( P + R )

where TP, FP, and FN represent specific items that are used to calculate the F1 score in the context of a Confusion_matrix. In particular, when computing the micro-F1 score, TP corresponds to the number of predicted tuple that match exactly with those in the gold set.

The metric in different settings.

It is worth noting that our evaluation metrics consider two different settings.

In the common setting, the NER task only needs to extract the corresponding entity and determine its type; the head and tail entities predicted by the RE task can be replaced by their mentions.

In the strict setting, the NER task should give all the descriptions of the entity in the paragraph; the prediction results of the RE task should be strictly consistent with the Ground True.

Finally, we use the following score for ranking:

Scorecommon=0.5*Task1_common+0.5*Task2_common

Scorestrict=0.5*Task1_strict+0.5*Task2_strict

overall_score=0.5* Scorecommon +0.5* Scorestrict
